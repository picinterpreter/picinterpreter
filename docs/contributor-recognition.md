# 图语家贡献者致谢 · Contributor Recognition

图语家是一个公益开源 AAC 项目。我们目前没有稳定资金来源，但会认真记录和感谢真实贡献。

这份文档说明：

- 哪些贡献会被公开致谢
- 会用什么方式致谢
- 如何确认你的名字或署名方式
- 哪些贡献更适合匿名记录

## 我们感谢哪些贡献

以下都算真实贡献，不限于写代码：

- 提交代码、测试、修 bug、性能优化
- 改进 README、贡献指南、架构文档、研究笔记
- 整理 AAC 词库、图片来源、授权信息
- 提供真实照护沟通场景、匿名反馈、试用记录
- 做 UI/UX、无障碍、康复专业建议
- 帮助新贡献者理解 issue、拆分任务、补 acceptance criteria
- 翻译、校对、截图、演示视频、教程

## 致谢方式

我们当前采用以下几种方式：

### 1. GitHub 公开记录

适用于：

- 合并的 PR
- 被 issue / commit / doc 明确引用的公开贡献

公开记录包括：

- Git 提交历史
- PR / issue 关联记录
- 文档中的贡献者名单

### 2. README / 文档中的贡献者名单

适用于：

- 持续贡献者
- 完成较完整模块或重要文档的人
- 长期帮助整理资料、测试、用户研究的人

默认会记录：

- GitHub 用户名
- 可选的名字或署名
- 贡献方向，例如：前端、测试、文档、词库、用户研究、康复反馈

### 3. 主题致谢

适用于：

- 提供一批高价值匿名反馈的人
- 协助组织试用、访谈、招募的人
- 提供专业意见的康复老师、照护者、志愿者

这类情况可以只写成群体致谢，例如：

- “感谢 2026 年 5 月参与匿名照护者反馈的家庭”
- “感谢帮助审阅词库优先级的康复从业者志愿者”

### 4. 推荐信 / 项目经历说明

对持续贡献且需要求学、求职、申请材料支持的贡献者，维护者可以在了解实际贡献后，尽量提供：

- 贡献内容说明
- 项目背景说明
- 推荐语或简短证明

这类支持不是自动承诺，但我们愿意认真对待。

## 默认署名规则

如果你没有特别说明，默认按以下规则处理：

- 代码 / 文档 PR：使用 GitHub 用户名
- Issue 中的公开贡献：使用 GitHub 用户名
- 私下提供的反馈：默认不公开姓名，除非你明确同意

如果你希望调整署名方式，可以在 PR、issue，或联系维护者时注明：

- 希望显示的名字
- 是否希望附上 GitHub 链接
- 是否接受写入公开致谢页
- 是否只接受匿名致谢

## 匿名与隐私

以下内容默认不公开：

- 患者姓名
- 家属姓名
- 电话、地址、病历、医院信息
- 患者照片、声音、家庭照片
- 能直接识别个人身份的详细故事

如果你的贡献来自真实患者或家庭场景，请优先使用匿名方式。我们更看重反馈本身，而不是公开署名。

## 当前建议的致谢分组

后续可以按下面的结构维护公开致谢页：

### Core Maintainers

- 负责主要架构、方向和合并维护的人

### Code Contributors

- 提交代码、测试、修复和功能的人

### Documentation and Research Contributors

- 编写文档、研究资料、整理参考的人

### Design and Accessibility Contributors

- 提供视觉、交互、无障碍建议的人

### Caregiver and Clinical Feedback

- 以匿名或群体形式感谢提供真实场景反馈的家属、照护者、康复从业者

## 建议的公开致谢页模板

后续如果需要把致谢名单直接挂到 README 或单独页面，可按下面格式维护：

```md
## Contributors

- `@username` — frontend, receiver flow, tests
- `@username2` — documentation, onboarding, screenshots

## Special Thanks

- Anonymous caregivers who shared real-world communication scenarios
- Rehabilitation and speech-language professionals who reviewed priority vocabulary
```

## 如何申请补记致谢

如果你已经贡献过，但没有被记录，可以：

1. 在相关 PR / issue 下留言补充
2. 新开一个 docs / meta issue
3. 直接提交一个小 PR 更新致谢页

我们会优先接受“事实清楚、范围明确”的补记。

## 当前原则

- 致谢应尽量真实、可核对、不过度包装
- 不把患者或家属隐私暴露为“故事素材”
- 不把一次很轻的互动夸大成长期贡献
- 也不忽视文档、测试、反馈、研究这类非代码贡献

如果你帮助了图语家推进，我们希望这份记录能让你的付出被认真看见。
