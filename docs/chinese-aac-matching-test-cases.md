# 中文 AAC 图符匹配测试集

这个文档用于记录图语家在“文字/词语/句子 -> 图片/图符”匹配时容易出错的案例。

重点不是一次性做完整，而是持续收集：

- 哪些输入容易匹配错图
- 用户实际期望什么图
- 错误原因是什么
- 修复后不能再次出错

## 测试表

|---|---|---|---|---|---|---|---|---|
| 1 | 开心 | 情绪表达 | 开心表情、笑脸 | 开心果 | 错误 | 子串误匹配 | P1 | “开心”不应匹配到“开心果” |

## 错误类型

| 错误类型 | 含义 | 例子 |
|---|---|---|
| 子串误匹配 | 短词被匹配进更长词 | 开心 -> 开心果 |
| 多义词 | 一个词有多个意思 | 苹果：水果 / 品牌 |
| 抽象词 | 很难直接用图片表达 | 难受、喜欢、害怕 |
| 动作 vs 物品 | 系统不知道该选动作还是物体 | 喝水：喝的动作 / 水杯 |
| 否定句 | 忽略“不、不要、不能”等否定词 | 不想吃饭 -> 吃饭 |
| 上下文未消歧 | 句子里本来有线索，但系统没用上 | 我很开心 -> 开心果 |
| 医疗高风险 | 错图可能影响照护判断 | 头疼、胸闷、疼痛 |

## 优先级

| 优先级 | 标准 |
|---|---|
| P0 | 医疗、疼痛、危险、紧急需求相关，错图可能造成照护风险 |
| P1 | 高频日常沟通需求，错图会明显影响使用 |
| P2 | 情绪、社交、一般生活表达 |
| P3 | 低频词、边界情况、长尾问题 |

## 后续用途

这些案例后续可以用于：

- 修复图符元数据，比如 `aliases`、`negativeHints`
- 建立自动化测试，避免修好后再次出错
- 给贡献者提供明确任务
- 作为真实用户测试前的基础质量检查
