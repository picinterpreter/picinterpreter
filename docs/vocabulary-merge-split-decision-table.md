# 图语家词汇合并/分开决策表（V1）

本文档回答一个很实际的问题：

- 哪些同义词或意思相近的词，应该合并成一个概念
- 哪些词虽然接近，但应该拆开成不同概念
- 当前图语家的 V1 词表和 `seed` 数据，已经做到哪里，还差什么

这份文档不是字典学规则，而是**AAC 沟通后果导向**的决策表。

## 一句话原则

不要先问“这两个词像不像”，要先问：

**如果患者说 A 和说 B，会不会让照护者做出不同理解、不同回应、不同动作。**

- 会：倾向分开
- 不会：可以先合并
- 不确定：先做“主概念 + 同义入口”，等真实使用数据再决定

---

## 判断规则

### 1. 只是不一样的叫法：合并

适用条件：

- 主要差异是口语习惯
- 指向的是同一个对象/地点/人物
- 共用同一张图不会明显误导

例子：

- `医生` / `大夫`
- `厕所` / `洗手间` / `卫生间`
- `开心` / `高兴` / `快乐`

推荐做法：

- 一个主概念
- 其余词做 `synonym / alias`

### 2. 会导致不同照护动作：分开

适用条件：

- 听到词后，对方应该做的事不同
- 所属场景/板块不同
- 一张图无法稳定表达两者

例子：

- `手机` / `打电话`
- `药` / `吃药`
- `医生` / `医院` / `叫医生`

推荐做法：

- 拆成不同概念
- 必要时分别进不同板块

### 3. 强度、紧急度、求助后果不同：倾向分开

适用条件：

- 虽然语义接近，但会影响照护优先级
- 患者说出来时，家属/护士对严重程度的判断会变

例子：

- `难受` / `不舒服`
- `痛` / `很痛`

推荐做法：

- 系统内部先分开
- 患者端是否都高频显示，可以后面再简化

### 4. 高风险抽象词：固定板位，不靠搜索

适用条件：

- 抽象
- 高频
- 语义依赖上下文

例子：

- `想`
- `要`
- `喜欢`
- `痛`

推荐做法：

- 保留成独立核心词
- 放稳定板位
- 不靠开放搜索临时决定

### 5. 广义词与具体词并存：保留两层

适用条件：

- 既需要一个泛词快速表达
- 又需要更具体的下钻

例子：

- `痛` 与 `头痛` / `肚子疼`
- `难受` 与 `恶心` / `头晕`

推荐做法：

- 保留广义词作为第一层入口
- 逐步补具体症状词作为第二层

## 商业 AAC 的常见优化方式

根据前面调研，成熟产品通常不是靠“词典语义最完美”来解决这个问题，而是靠下面几种组合：

1. 高频核心词固定位置
2. 身体/医疗/厕所/修复单独成板
3. 人物、物品、动作分开
4. 同叫法合并，不同行动分开
5. 模糊词放到明确语境板里，少靠自由搜索

换句话说，商业 AAC 优化的是：

**沟通后果的稳定性**，不是词义分析的纯理论完备性。

## 决策表

下面这张表是图语家 V1 最值得先明确的一批词组。

| 词组/近义组 | 建议 | 原因 | 商业 AAC 常见做法 | 当前词表状态 | 结论 |
|---|---|---|---|---|---|
| `厕所 / 洗手间 / 卫生间` | 合并为一个主概念 | 都指向同一个地点/需求入口 | 通常固定在 toilet / hygiene 板 | `p_toilet` 已吸收这些词 | 已对齐 |
| `上厕所 / 去厕所` 与 `厕所` | 主概念 + 同义入口 | 患者常说的是需求，不一定是地点浏览 | 通常做成高频快捷入口 | `上厕所` 已进 `p_toilet` synonym；`我想去厕所` 也有快捷语 | 已对齐 |
| `医生 / 大夫` | 合并为一个主概念 | 只是叫法差异 | doctor 常保留单一人物概念 | `p_doctor` 已吸收 `大夫` | 已对齐 |
| `医生 / 医院 / 看病 / 叫医生` | 分开 | 人、地点、行为、求助动作不同 | 不同板块/不同按钮 | `医生` 与 `医院` 已分开；`看病` 现在归 `医院`；`请叫医生` 独立存在 | 基本对齐 |
| `手机 / 电话` | 主概念 + 同义入口 | 在中文里常被当作同一物品概念 | 物品概念通常合并 | `p_phone` 保留 `电话` | 已对齐 |
| `手机 / 打电话` | 分开 | 物品 vs 动作 | 商业 AAC 常分 object 与 action | `p_phone` 与 `p_make_call` 已分开 | 已对齐 |
| `药 / 药物` | 合并为一个主概念 | 只是叫法差异 | medicine 常是单一对象概念 | `p_medicine` 已吸收 `药物` | 已对齐 |
| `药 / 吃药 / 服药` | 分开 | 物品 vs 行为/需求 | commercial AAC 常分 medicine 与 take medicine | `p_medicine` 与 `p_take_medicine` 已分开 | 已对齐 |
| `开心 / 高兴 / 快乐` | 主概念 + 同义入口 | 都是正向情绪词 | feelings 板常合并为一个情绪入口 | `p_happy` 已吸收 | 已对齐 |
| `不开心 / 难过 / 伤心 / 悲伤` | 主概念 + 同义入口 | 都是负向情绪，但 V1 不需要拆太细 | feelings 板常用单一负向情绪入口 | 当前 `p_sad` 用 `伤心` 承接 `不开心 / 难过 / 悲伤` | 可接受 |
| `难受 / 不舒服` | 倾向分开 | 强度与主观感受不同，容易影响照护判断 | 模糊症状词常拆开并放医疗板 | 当前已拆成 `p_uncomfortable` / `p_unwell` | 已对齐 |
| `难受 / 痛` | 分开 | “难受”不一定等于疼痛 | pain 常与 broader discomfort 分离 | `难受` 已从 `p_pain` synonym 移除 | 已对齐 |
| `不舒服 / 生病` | 分开 | 不舒服是状态；生病是更明确判断 | 商业 AAC 不会把 unwell 全并入 sick | `不舒服` 已从 `p_sick` synonym 移除 | 已对齐 |
| `恶心 / 想吐 / 反胃` | 主概念 + 同义入口 | 同一症状簇 | 医疗板里常合并 | 当前 `p_nausea` 已吸收 `想吐 / 反胃` | 已对齐 |
| `痛 / 头痛 / 肚子疼 / 胸口疼` | 广义词 + 具体词并存 | 既要快速表达，也要更细致定位 | pain + body-part 是成熟做法 | `p_pain` 与 `p_headache / p_stomachache / p_chest_pain` 已并存 | 已对齐 |
| `想 / 要` | 分开保留 | 都是高频核心词，但句法功能不同 | 核心词通常各自占位 | `p_want(想)` 与 `p_想要(要)` 已收紧边界 | 已对齐 |
| `喜欢 / 爱` | 主概念 + 同义入口 | V1 可先不细分偏好强度 | feelings/core 常合并 | 当前 `p_love(喜欢)` 吸收 `爱` | 可接受 |

## 当前词表核对结果

这里的“当前词表”包含两层：

- [V1 词表草案](/D:/used-by-codex/picinterpreter/picinterpreter-github/docs/tuyujia-v1-initial-concept-list.md)

### 已经比较一致的部分

这些地方已经明显体现了前面研究出来的方向：

- `厕所 / 洗手间 / 卫生间 / 上厕所`
  - 已合并到 `p_toilet`
- `医生 / 大夫`
  - 已合并到 `p_doctor`
- `医生 / 医院 / 请叫医生`
  - 已拆开
- `开心 / 高兴 / 快乐`
  - 已合并到 `p_happy`
- `难受 / 不舒服 / 生病 / 痛`
  - 现在已经不再全部混成一个词
- `手机 / 打电话`
  - 已拆成 `p_phone` 和 `p_make_call`

### 已经朝对的方向走，但还是过渡态的部分

这些不是错误，但还没达到最清晰的 V1 结构。

#### 1. `治疗师 / 看护人`

当前状态：

- `治疗师` 与 `看护人` 已补入默认词库

问题：

- 目前已经可用，但还没有更多场景词去细化不同照护关系

建议：

- V1 先保持这样
- 后续如果真实使用里常出现，再考虑是否补：
  - `言语治疗师`
  - `康复治疗师`
  - `护工`

#### 2. `恶心` 的图片仍是过渡态

当前状态：

- `p_nausea` 已不再是空白图
- 但现在仍是“能显示的真实 AAC 医疗图”，不是最理想的专用症状图

建议：

- 后续如果找到更精准的 `恶心 / 想吐` 图，再替换

### 当前还需要注意的一处过渡实现

#### `恶心` 的图片

当前状态：

- `p_nausea` 已经不再是空白图
- 但目前还是过渡性医疗图，不是最理想的专用症状图

建议：

- 后续如果找到更贴切的“恶心/想吐”图，再换成更精准的症状图
- 但从“不要空白卡片”的角度，它已经比之前稳很多

## 面向 V1 的实际结论

如果按成熟产品思路来收束，图语家 V1 可以把这些词分成三类：

### A. 直接合并成一个主概念

- `医生 / 大夫`
- `厕所 / 洗手间 / 卫生间`
- `开心 / 高兴 / 快乐`
- `恶心 / 想吐 / 反胃`

### B. 一个主概念 + 若干同义入口

- `上厕所 -> 厕所`
- `电话 -> 手机`
- `不开心 / 难过 / 悲伤 -> 伤心`

### C. 必须拆开

- `手机` / `打电话`
- `药` / `吃药`
- `医生` / `医院` / `叫医生`
- `难受` / `不舒服`
- `难受` / `痛`

---

## 现在最值得继续优化的点

按优先级看，后面最值得继续做的是：

1. 后续再替换更准确的 `恶心` 图片
2. 观察 `治疗师 / 看护人` 是否需要更细分
3. 如果真实使用中出现，再继续补更具体的症状短语

---

## Bottom Line

图语家现在已经不再是“词一多就全混一起”的状态了。

按照前面商业 AAC 的优化逻辑看，当前 `seed` 词库已经开始接近下面这个方向：

- 同叫法合并
- 不同行动分开
- 不同后果分开
- 模糊词放进医疗/需求/修复等明确板块

目前最大的剩余工作，不是再发明更多理论，而是继续把这套决策：

- 落到 `seed`
- 落到测试
- 落到板块结构

这样后面再迭代图库时，才不会慢慢退回“词图又混了”的状态。
