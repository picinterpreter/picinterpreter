# AGENTS.md

## Product Context

图语家是给失语症患者和照护者使用的图片辅助沟通工具。主界面的第一用户是可能无法顺利阅读、理解长句说明或抽象文字解释的患者；照护者相关的设置、导入、调试页面可以承载更多文字，但也应保持清晰、克制。

## UI/UX Rules

- 患者主流程优先使用图片、图标、颜色分组和直接动作表达含义，避免依赖文字说明。
- 不要在主界面、表达流程、接收展示、紧急求助、播放覆盖层中加入小字说明、长段帮助文字或操作说明。
- 如果某个提示确实必要，优先换成可识别的图标、状态形状、进度反馈或按钮状态；文字只保留最短动词或名词，例如“表达”“接收”“完成”“重播”。
- 按钮文案要短、直接、可行动，不写解释性句子。避免“点击此处以……”这类说明式文案。
- 对患者可见的错误状态应尽量用高可见度状态和简短文字表达，例如“无法播报”“未找到”，不要展示技术性错误细节。
- 照护者区域可以有说明文字，包括设置、账号、导入、调试、同步状态等，但要和患者主流程分层，不要挤进患者正在使用的沟通界面。
- 紧急求助界面必须保持零学习成本：大按钮、强对比、少文字、一次点击即可播报。
- 图标应使用项目统一的线性图标组件或现有设计系统图标；避免新增散乱 emoji 作为功能图标。分类数据自带图标可以保留作为分类识别标记。
- 保持触控目标不小于 44px，关键患者操作要更大，并留足间距，避免误触。
- 视觉风格继续靠近 Apple 系列：浅灰系统背景、克制的半透明面板、胶囊控件、轻阴影、清晰焦点态，不使用花哨装饰。

## Implementation Notes

- 不删除已有功能来换取视觉简化；需要隐藏复杂信息时，应移动到照护者设置、抽屉、调试页或渐进展示区域。
- 修改 UI 后必须运行 `npm run build`，并确保构建成功。
- 涉及患者主流程的 UI 改动，需要额外检查移动端小屏下是否存在文字拥挤、按钮太小、控件重叠。
